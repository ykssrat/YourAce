二、当前建议的迭代重点（面向 0.1.6）

- 收敛前端策略入口，只暴露已实现或明确标注状态的子策略
- 清理评分器残留文案，统一为“看法矩阵 + 子策略调度”表述
- 让 Lean 回测与现有子策略形成最小验证闭环
- 完善策略加载失败、数据不足等场景的接口与前端提示
- 将启动期知识卡片与品牌视觉主题从“已实现最小版”继续打磨为正式启动页

三、版本说明

- 当前代码已完成从评分器主导向子策略矩阵主导的架构过渡
- `score` 字段已从核心接口与联调脚本中移除，主表达已切换到 `label + horizon_signals + matrix`
- 移动端已落地最小启动页：按日期轮换知识卡片，并在启动阶段进行本地设置读取与健康检查预热，且至少停留 2 秒
- 下一阶段重点不是继续堆功能，而是补足策略可解释性、验证链路、视觉统一与文档一致性
