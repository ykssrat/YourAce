# 子策略架构设计：全天候 3x3 看法矩阵 (Opinion Matrix)
**架构：3x3 看法矩阵**
每个子策略输出一个 $3 \times 3$ 的矩阵，覆盖三个时间维度和三个方向。

| 周期 \ 方向 | 看多 (+1) | 观望 (0) | 看空 (-1) |
| :--- | :---: | :---: | :---: |
| **短期 (Short-term)** | $V_{s}$ | - | - |
| **中期 (Mid-term)** | - | $V_{m}$ | - |
| **长期 (Long-term)** | - | - | $V_{l}$ |
##  拟引入的等效接口 (Sub-strategy Instances)
| 策略接口 | 主要关注点 | 矩阵特征 |
| :--- | :--- | :--- |
| **利弗莫尔策略 (Livermore)** | 趋势突破、关键点位 | 中短期倾向明显，追随强趋势 |
| **机器学习策略 (ML)** | 模式识别、非线性相关 | 全周期输出，依据模型置信度投票 |
| **MACD 策略** | 动量交叉、背离 | 典型的短期/中期趋势指标 |
| **价值投资策略 (Value)** | 基本面、估值锚点 | **主要投票长期**，忽略短期波动 |
| **KDJ 策略** | 超买超卖、随机波动 | 极度关注短期超导 |
| **信息面策略 (Sentiment)** | 新闻、研报、情绪 | 捕捉短期爆发力及中长期逻辑转变 |

##  全天候战略实现路径

### A. 聚合算法 (The Consensus Engine)
通过一个“元策略” (Meta-Strategy) 收集所有子策略的矩阵，进行加权投票：
- **一致性检查**：如果长短期看法不一，采取“风险对冲”或“分仓操作”。
- **风格动态加权**：根据当前市场环境（Regime），动态调整利弗莫尔（趋势）和价值（均值）的权重。

### B. 上线 App 路径
1. **模块化重构**：在 `Algorithm.Python/SubStrategies` 下建立文件夹。
2. **中间层适配**：编写一个 `MatrixToInsight` 适配器，将 3x3 矩阵转化为 Lean 引擎可识别的 `Insight` 对象。
3. **可视化仪表盘**：在 App 端展示“策略委员会”的投票情况，而不仅仅是一个最终的买卖信号。

---

> [!TIP]
> 这种设计的本质是**将“预测”降维为“共识”**。当多个不同维度的策略在特定周期达成共识时，交易的胜率将大幅提升。
