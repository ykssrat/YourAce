YourAce 诊股信息可信度改造方案（仅方案，不改代码）

一、背景问题

当前诊股页有三个明显问题：

1. 标的名称不清晰
- 只输入代码时，页面未稳定展示“代码 + 名称 + 资产类型”，用户无法快速确认是否诊断了目标标的。

2. 资讯可信度不足
- 资讯区域常出现明显不真实内容，用户会直接失去信任。
- 说明资讯抓取链路没有“可验证来源 + 失败降级策略 + 真实性标记”。

3. ETF/场外基金场景不匹配
- ETF 或场外基金并不适合展示“公司新闻”。
- 更有价值的是展示持仓结构（前十大持仓及占比、行业分布、重仓股集中度）。

二、改造目标

1. 诊股首屏必须确认标的身份
- 诊断结果卡片第一行固定显示：代码、名称、资产类型。

2. 资讯区改为“可信来源 + 可验证”
- 只展示可回溯来源的资讯。
- 每条资讯保留来源站点、发布时间、原文链接。

3. ETF/基金按品类切换信息面板
- 股票：展示公司资讯。
- ETF/场外基金：默认不展示公司资讯，改展示持仓结构。

三、信息架构调整（UI）

诊股页建议分为四块：

1) 标的确认卡
- 代码
- 名称
- 资产类型（股票/ETF/场外基金）
- 数据时间戳

2) 评分与矩阵卡
- 总分、总体标签
- 3x5 看法矩阵

3) 动态信息卡（按资产类型切换）
- 股票：最新资讯（真实来源）
- ETF/基金：持仓结构（Top holdings）

4) 解释与风险卡
- 选中特征
- 关键提示（如：资讯不可用、持仓数据延迟）

四、真实资讯方案（股票）

1. 数据源策略

优先级建议：
- 一级源：交易所公告/基金公司公告/上市公司公告接口
- 二级源：主流财经媒体开放接口（需许可）
- 三级源：聚合源（仅做补充，必须带来源字段）

2. 数据质量规则

每条资讯必须包含：
- title
- publisher
- published_at
- source_url
- source_type（official/media/aggregator）

过滤规则：
- 标题为空或长度异常：丢弃
- 无来源链接：丢弃
- 发布时间超窗（如超过 30 天）：默认折叠
- 相似标题去重（避免多次重复）

3. 失败降级

当资讯不可用时，不显示“假资讯”，而是显示明确状态：
- 文案："当前未获取到可验证资讯"
- 操作："点击重试"
- 附加：显示最后一次成功更新时间

4. 最小可用接口建议

建议新增统一资讯接口（仅方案）：
- GET /news/verified?code=...&asset_type=stock&limit=...

返回字段建议：
- code
- name
- items[]: {title, publisher, published_at, source_url, source_type, confidence}
- fetch_status: ok/empty/error
- fetched_at

五、ETF/场外基金持仓展示方案

1. 展示什么

优先展示：
- 前十大持仓（证券代码、名称、权重）
- 前十大合计占比
- 行业分布（可选）
- 持仓更新时间

2. 不展示什么

- 不展示“公司资讯流”作为默认内容
- 不混入与基金无关的个股新闻

3. 失败降级

- 若持仓数据暂不可得：显示"当前未获取到持仓结构"，并保留刷新按钮
- 若数据过旧：显示"持仓数据可能滞后（日期）"

4. 最小可用接口建议

建议新增持仓接口（仅方案）：
- GET /holdings?code=...&asset_type=etf|fund

返回字段建议：
- code
- name
- asset_type
- as_of_date
- top_holdings[]: {holding_code, holding_name, weight}
- top10_weight_sum
- industry_distribution[]（可选）

六、前端交互规则（关键）

1. 诊断按钮点击后，先渲染“标的确认卡”
- 若拿不到 name，不进入结果区，提示“未识别标的名称”。

2. 动态信息区按 asset_type 路由
- stock -> 资讯卡
- etf/fund -> 持仓卡

3. 展示可信度标记
- 每条资讯显示来源与时间
- 对聚合源标记“二级来源”

七、接口兼容建议

为避免一次改动过大，采用两阶段：

阶段 1（快）：
- 先在现有诊股响应中补 name、asset_type
- 资讯只保留可验证字段，不合规即隐藏

阶段 2（稳）：
- 新增 /news/verified 与 /holdings 独立接口
- 前端按资产类型切换请求

八、验收标准

1. 用户输入代码后，页面 1 秒内能看到“代码 + 名称 + 类型”。
2. 股票资讯至少 90% 带来源链接和发布时间。
3. ETF/基金诊股页面默认展示持仓结构，不展示公司资讯流。
4. 当数据不可得时，页面显示明确状态，不出现“疑似伪造内容”。
