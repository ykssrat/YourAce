YourAce 选股与诊股功能设计文档

一、文档目标

本文档定义应用的两项核心功能如何设计：

- 选股
- 诊股

目标做两件事：

1. 根据筛子条件输出候选代码
2. 根据理财产品类型与代码输出诊断矩阵


二、功能范围

1. 选股

输入筛选矩阵：

- 理财产品类型：股票 / ETF / 场外基金
- 窗口期：短 / 中 / 长
- 分数条件：大于某值 / 小于某值
- 看法：强烈看多 / 看多 / 观望 / 看空 / 强烈看空

输出结果：

- 每轮实验 50 个候选标的
- 返回符合条件的代码列表与核心结果卡片

2. 诊股

输入条件：

- 理财产品类型
- 代码

输出结果：

- 总分
- 总体标签
- 窗口期诊断结果
- 看法矩阵

三、APK 信息架构

建议底部双 Tab 结构：

1. 选股
2. 诊股

推荐理由：

- 两个功能目标完全不同
- 选股强调批量筛选
- 诊股强调单代码解释
- 双 Tab 比单页堆叠更清晰

四、选股页面设计

1. 页面目标

用户希望快速输入筛子条件，跑一轮，得到一批可进一步研究的代码。

2. 页面布局

从上到下建议这样排：

A. 顶部标题区

- 标题：选股
- 副标题：按产品类型、窗口期、分数和看法筛选候选代码

B. 筛选区

建议使用分组卡片样式，每组一行或两行，不要塞进一个超长表单。

字段如下：

- 产品类型
  - 股票
  - ETF
  - 场外基金

- 窗口期
  - 短
  - 中
  - 长

- 分数条件
  - 大于
  - 小于

- 分数值
  - 建议输入框或滑条
  - 默认值可设为 60

- 看法
  - 强烈看多
  - 看多
  - 观望
  - 看空
  - 强烈看空

- 每轮实验数
  - 固定展示 50
  - 初期不开放修改，保持流程稳定

C. 操作区

- 主按钮：开始选股
- 次按钮：重置条件
- 状态提示：当前是否使用缓存、是否跳过 stock_list 更新

D. 运行反馈区

用于显示当前批处理进度：

- 当前轮次
- 当前已扫描数量
- 缓存命中数量
- 新算代码数量
- 评分达标数量
- 信号不匹配数量

E. 结果列表区

每个结果用卡片展示，建议字段：

- 代码
- 名称
- 产品类型
- 总分
- 总体标签
- 窗口期标签
  - 短
  - 中
  - 长
- 进入诊股按钮
- 加入自选按钮

3. 选股页面交互建议

- 用户点“开始选股”后，筛选区折叠为摘要栏，避免遮挡结果
- 结果先出先展示，不必等整轮全部完成后才刷新
- 若无命中，页面给出明确解释：
  - 没有符合条件的标的
  - 当前原因可能是分数阈值过严或窗口期条件过严
  - 推荐放宽建议，例如：把分数从 80 降到 60

4. 选股页面默认策略建议

短线买入场景默认值建议：

- 产品类型：股票
- 窗口期：短
- 分数条件：大于
- 分数值：60
- 看法：看多

原因：

- 这更接近短线买点，而不是强趋势共振股
- 80 分以上适合波段/中线，不适合作为短线默认门槛

五、诊股页面设计

1. 页面目标

用户已经有一个明确代码，希望查看当前值不值得买、处于哪个窗口期状态，以及总体解释。

2. 页面布局

A. 顶部标题区

- 标题：诊股
- 副标题：输入代码查看三窗口诊断与总体分数

B. 输入区

- 产品类型选择
  - 股票 / ETF / 场外基金
- 代码输入框
- 搜索按钮或诊断按钮
- 历史记录下拉
- 自选快捷入口

C. 总览卡片

建议展示：

- 代码 + 名称
- 总分
- 总体看法标签
- 更新时间

D. 矩阵卡片

这是诊股页面最重要的部分。

六、诊股矩阵设计

你提出的矩阵是：

- 窗口期：短 / 中 / 长
- 看法：强烈看多 / 看多 / 观望 / 看空 / 强烈看空

建议用 3 x 5 矩阵展示。

1. 矩阵结构

行：

- 短
- 中
- 长

列：

- 强烈看多
- 看多
- 观望
- 看空
- 强烈看空

2. 展示方式

每一行只高亮一个格子，表示当前窗口期落在哪个看法桶里。

例如：

- 短期：看多
- 中期：观望
- 长期：强烈看多

则矩阵中对应三个格子高亮，其余灰置。

3. 为什么这样设计

- 用户一眼就能看出三个窗口期是否一致
- 比直接显示 short/mid/long 文本更直观
- 与“诊股”功能的解释型定位更匹配

4. 总分与矩阵的关系

建议在矩阵卡片顶部增加一行说明：

- 总分是短中长期信号加权后的综合结果
- 矩阵是每个窗口期单独的看法定位

也就是：

- 总分负责总结
- 矩阵负责拆解

七、诊股页面的附加信息

在矩阵下方增加两个解释区：

1. 结构解释

- 短期是否偏强
- 中期是否拖累
- 长期是否提供方向支撑

2. 辅助信息

- 选中特征数
- 选中特征列表
- 是否开启新闻
- 最新新闻摘要（可折叠）

八、交互流程建议

1. 选股 -> 诊股

- 用户在选股页命中某个代码后
- 点“进入诊股”
- 自动带入产品类型与代码
- 跳转诊股页

2. 诊股 -> 自选

- 用户在诊股页查看后
- 可以加入自选
- 后续在首页或自选页快速回看

九、页面样式建议

1. 设计方向

- 不做交易软件那种堆表格风格
- 用卡片化信息层级提升可读性
- 页面底色保持现有浅绿系即可，延续当前原型

2. 颜色建议

看法颜色建议固定：

- 强烈看多：深绿
- 看多：绿色
- 观望：橙色
- 看空：红色
- 强烈看空：深红

3. 关键视觉重点

- 选股页重点突出结果数量和代码卡片
- 诊股页重点突出总分和 3x5 看法矩阵

十、后续接口建议

为了支持这两个 UI，后端建议补两类接口：

1. 选股接口

建议路径：

- POST /screen

建议请求字段：

- asset_type
- horizon
- score_operator
- score_threshold
- opinion
- round_size
- max_rounds
- use_cache

建议响应字段：

- items
- scanned_count
- cache_hit
- cache_miss
- score_pass_count
- signal_miss_count

2. 诊股接口

建议路径：

- POST /diagnose

建议请求字段：

- asset_type
- code

建议响应字段：

- code
- name
- score
- label
- matrix
- horizon_signals
- selected_features
- latest_news

十一、第一阶段实现优先级

建议先做最小可用版本：

第一阶段：

- 选股 Tab
- 诊股 Tab
- 选股筛子表单
- 诊股总分卡片
- 诊股 3x5 看法矩阵
- 选股结果跳转诊股

第二阶段：

- 自选联动
- 历史诊断记录
- 新闻折叠区
- 结果排序与二次筛选


优先做成：

- 一个真正可用的选股页
- 一个真正可解释的诊股页
