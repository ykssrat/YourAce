YourAce 版本整理（0.1.5）

版本号：0.1.5
整理日期：2026-03-19

一、本版本目标

- 从“单评分器输出”过渡到“可切换子策略 + 统一看法矩阵”的新架构。
- 为后续工业级回测与策略扩展引入 Lean 工程底座。
- 收敛移动端与后端的策略接口，减少版本发布中的构建噪音。

二、核心新增与调整

1) 子策略机制正式上线

- 后端 `analyze`、`screen`、`diagnose` 请求统一支持 `strategy` 参数。
- `opinion_engine` 改为调度器角色，根据策略名动态分发到具体策略模块。
- 当前已接入：
  - `momentum_deviation`
  - `livermore`
- 策略显示名与内部 ID 做了兼容映射，便于移动端直接调用。

2) 评分器淡出，统一回到看法矩阵

- API 返回结构保留 `score` 字段以兼容旧端，但值固定为 `0.0`。
- `label`、`horizon_signals`、`matrix` 成为当前核心输出。
- 这意味着系统判断重心已从单分数排序，转向短中长周期的结构化看法表达。

3) 移动端增加策略切换能力

- 选股页与诊股页已具备策略选择入口。
- 前端类型定义中补充多种策略选项，为后续继续接入新策略预留 UI 位。
- App 与后端的请求模型完成对齐，策略切换不再需要额外适配。

4) Lean 工程进入仓库

- 新增 `scripts/backtest/lean` 目录，作为工业级回测与研究底座。
- 文档层开始明确“轻量研究引擎 + Lean 工业引擎”的双引擎结构。
- 版本方向从“可演示”进一步转向“可验证、可扩展、可迁移”。

5) 发布物治理与仓库清理

- 停止继续提交 Android `.cxx` 等编译中间产物。
- `.gitignore` 补充忽略规则，降低无效差异与仓库噪音。
- `build.gradle`、`package.json`、API version 已同步提升到 `0.1.5`。

三、对外能力变化

1) 后端能力

- API 标识版本升级为 `0.1.5`。
- 面向调用方的核心变化不是新增接口，而是接口语义升级：
  - 支持策略选择
  - 输出以看法矩阵为中心
  - 为多策略并存留出稳定入口

2) 移动端能力

- 用户可在界面上切换不同策略后再执行选股或诊股。
- 当前前端策略枚举已先行扩容，但后端实际稳定接入的仍以 `momentum_deviation` 与 `livermore` 为主。

3) 研发能力

- Lean 工程入库后，YourAce 不再只是一套轻量打分与展示工具，而是开始具备研究和工业验证的双轨演进条件。

四、架构意义

- `0.1.5` 是一次明显的架构转向版本。
- 它弱化了“单评分器主导一切”的旧路径，强化了“策略模块化 + 看法矩阵统一出口”的新路径。
- 这为后续引入状态路由、策略投票、回测验证和生产部署打下了接口基础。

五、已知边界

1) 前端策略枚举超前于后端实现

- 前端已经展示多种策略名，但后端并非都已具备对应实现。
- 若调用未实现策略，当前接口会报策略加载失败。

2) 文档仍有旧表述残留

- 现有部分文档仍沿用“评分器”叙述，与 `0.1.5` 实际架构并不完全一致。
- 后续需要逐步把“评分器中心”文案改写为“矩阵/子策略中心”。

3) Lean 已入库，但尚未与 API 主链路深度打通

- 当前 Lean 更多是研究与验证底座。
- 生产接口仍主要依赖本地轻量逻辑与策略模块输出。

六、对 0.1.6 的自然承接

- 把前端展示的策略枚举收敛到“已实现”和“规划中”两层，减少假可用项。
- 将 Lean 回测结果与当前子策略定义做映射，形成“研究 - 验证 - 输出”闭环。
- 清理旧评分器表述，补齐策略实现文档和失败提示。
