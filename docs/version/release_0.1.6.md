YourAce 版本前瞻（0.1.6）

版本号：0.1.6
整理日期：2026-03-19
文档性质：进行中

一、版本主题

- 从“子策略已可切换”推进到“子策略可解释、可验证、可扩展”。
- 把 `0.1.5` 的架构转向，落实成更稳的产品体验与研发闭环。

二、建议聚焦方向

1) 策略能力收敛

- 明确区分“已实现策略”和“占位策略”。
- 优先补齐 `momentum_deviation`、`livermore` 的说明文档、适用市场与信号特征。
- 若短期内不会实现马丁、KDJ、金叉死叉、多因子等策略，前端先隐藏或标注“即将开放”。

2) API 语义清理

- 当前状态：`score` 已从核心接口与主要联调脚本中移除，主判断字段已切换为 `label`、`horizon_signals` 与 `matrix`。
- 为策略加载失败、数据不足、回测不可用等场景补充更清晰的错误提示。
- 考虑新增策略元信息接口，例如返回：
  - 支持的策略列表
  - 每个策略的显示名
  - 是否稳定可用
  - 简短说明

3) Lean 与主链路联动

- 用 Lean 回测去验证已有子策略，而不是只把 Lean 当作静态资料库。
- 选定 1 到 2 个代表性策略做标准化回测样板。
- 把关键回测指标沉淀回文档，为后续策略上下线提供依据。

4) 移动端表达升级

- 在策略选择处增加说明文案，告诉用户不同策略偏向什么风格。
- 在诊股结果页补充“本次使用策略”与“信号解释”。
- 若策略不可用，前端直接给出可读提示，不把错误原样暴露给用户。

5) 文档体系重写

- 把“评分器”相关文档改写为“矩阵引擎 / 子策略调度 / 双引擎验证”表述。
- 新增一份总览文档，说明：
  - API 如何选择策略
  - 移动端如何消费策略结果
  - Lean 在体系中的角色

6) 启动期内容占位与预热加载

- 当前状态：移动端已接入启动页最小版本。
- 已实现内容：
  - 启动时展示“今日理财小知识”卡片
  - 卡片内容按日期轮换
  - 启动阶段并行读取本地设置并做一次健康检查预热
  - 启动页最少停留 2 秒
- 当前尚未实现：
  - 品牌图标接入
  - 正式 slogan 视觉稿
  - 手动跳过
  - 只在每日首次启动时展示
- 展示内容可采用“每日一个理财小知识”或“当日市场认知卡片”的形式，根据当前日期生成或选择若干条短内容。
- 该模块的产品目标不是增加交易建议，而是：
  - 在冷启动时提供可阅读内容，减少空白等待感
  - 为后台预启动应用关键进程、数据层或接口连通检查争取时间
  - 提升首次进入 App 时的完成度与陪伴感
- 内容应保持中性、科普、低风险，不直接构成投资建议。

三、0.1.6 待办清单

- 收敛策略入口，只展示已实现或明确标注状态的策略。
- 补充策略元信息与失败提示，降低“选了但不可用”的困惑。
- 推进 Lean 对现有子策略的最小验证闭环。
- 改写旧“评分器”文档，使文档口径与 `0.1.5+` 架构一致。
- 将启动期知识卡片模块从“已实现最小版”升级为正式品牌启动页。

四、启动期知识卡片模块工作步骤

1) 产品定义

- 明确该模块是“启动占位 + 预热缓冲”能力，不是资讯流，也不是投资推荐页。
- 当前代码状态：
  - 已实现最少停留 2 秒
  - 尚未实现跳过机制
  - 尚未实现“每天只展示一次”
- 后续需要继续定义是否首次启动必看、是否每天只展示一次。
- 约束内容风格为短文本、易理解、低风险、非荐股导向。

2) 内容设计

- 设计知识卡片的数据结构，例如标题、正文、标签、日期键、风险提示。
- 先准备一批静态内容作为种子库，按日期轮换或按日历映射展示。
- 如需增强体验，可补充“今天为什么看到这条”之类的弱解释文案，但不要过度复杂。

3) 启动流程编排

- 在 App 冷启动入口增加启动页或模态层。
- 卡片展示的同时，并行触发以下预热动作：
  - 读取本地设置
  - 初始化 API 基地址
  - 进行基础健康检查
- 当前尚未实现首页数据预加载。
- 当前预热失败不会阻塞进入首页，但仍可继续补充更细的超时与降级说明。

4) 前端实现

- 增加启动态状态机，区分“展示中”“预热中”“可进入首页”“失败降级”。
- 当前已支持自动进入首页。
- 当前尚未支持手动跳过。
- 若当日内容加载失败，回退到默认欢迎文案，不影响主流程。

5) 后端与数据协同

- 初版可完全本地化，不依赖后端动态生成。
- 若后续希望服务端下发内容，再补充单独接口，但不应成为 `0.1.6` 首发阻塞项。

6) 验证与验收

- 验证冷启动时首页首屏是否更平滑，用户是否不再面对空白等待。
- 验证在网络差、接口失败、无缓存等场景下，启动页是否仍能正常退出。
- 记录预热收益，例如首页首次可交互时间是否下降、健康检查成功率是否更稳定。

五、建议产出物

- 一份策略总览文档。
- 一份 API 策略能力说明。
- 一套最小化 Lean 回测样板及结果记录。
- 一版移动端策略选择与结果解释优化。
- 一版启动期知识卡片与预热加载方案说明。

六、预期效果

- 用户层面：知道自己选了什么策略，也知道结果为什么这样。
- 研发层面：新增策略时不再只是“把代码接进去”，而是有统一入口、统一验证和统一文档。
- 产品层面：YourAce 从演示型工具继续向“可持续演进的策略平台”靠拢。
- 体验层面：应用冷启动阶段已经不再只有等待，而是有内容承接和后台预热；下一步重点是补齐品牌图标与正式视觉稿。

七、版本前瞻

- `0.1.6` 最值得做的，不是一次性塞进更多策略，而是把 `0.1.5` 已经打开的架构空间真正站稳。
- 如果这一版把“可选策略、稳定接口、验证闭环、文档一致性”四件事做好，后面的 `0.1.7+` 才适合继续扩张策略数量与智能化程度。
