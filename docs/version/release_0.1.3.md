YourAce 版本整理（0.1.3）

版本号：0.1.3
整理日期：2026-03-15

一、本版本目标

- 完成移动端“选股 + 诊股”双页主流程。
- 提升本地联调可用性（地址持久化、连通性自检、端口冲突自恢复）。
- 优化候选池分布，降低“几乎全是 0 开头代码”的体感问题。

二、核心变更清单

1) 后端 API 能力扩展

- 新增 POST /screen：支持批量筛股分页返回。
- 新增 POST /diagnose：支持单标的诊断并返回 matrix（三维看法矩阵）。
- /screen 支持“不限”筛选：
  - asset_type 可为空（不限）
  - horizon 可为空（不限）
  - horizon 不限 + opinion 有值时，任一窗口匹配即通过
- /screen 增强候选重排：
  - 由简单前缀轮询升级为“轻量聚类 + 轮询抽样”
  - 特征包含代码前缀、名称关键词推断行业、代码区段

2) 资产池与分布优化

- 在资产加载层补充一批宽基/行业 ETF（510/512/515/516/588/159 等）。
- 合并策略改为“补充资产优先拼接”，避免在 limit 截断时被完全挤掉。
- 实测首轮候选前缀分布出现 0/3/5/6/9/1，非 0 开头占比明显提升。

3) 移动端交互升级

- App 主结构：选股 Tab + 诊股 Tab。
- 顶部设置栏支持：
  - 后端地址输入
  - 新闻开关
  - 新增“测试连接”按钮（检测 /health 并显示失败原因）
- 设置持久化：
  - 使用 AsyncStorage 保存后端地址与新闻开关
  - App 重启后自动恢复上次设置
- 选股页筛选体验优化：
  - 产品类型新增“不限”
  - 窗口期新增“不限”
  - 分数阈值改为可输入 0-100（不再只靠固定档位）

4) 集成测试脚本增强

- 新增 tests/integration/test_local_api_diagnose.py。
- 诊股脚本支持端口冲突自动换端口（8010 -> 8011/8012...）。
- 当服务启动失败时，输出子进程日志，减少“只看到超时”排障成本。
- 修复 horizon_signals 打印兼容性（兼容字符串结构）。

5) 打包与交付

- Android 输出 APK 文件名固定为 YourAce.apk。
- 已完成新版 APK 构建与输出。

三、当前可用能力（面向演示）

- 本地命令行：可跑 analyze、diagnose、recommend 全链路。
- 移动端：可筛股、可跳转诊股、可查看矩阵、可做网络连通测试。
- 网络联调：支持局域网地址、USB 反向代理、Cloudflare 隧道三种方式。

四、已知限制与后续建议

1) 行业均匀分布当前为“名称关键词近似”

- 现有 stock_list 仅含 code/name，缺少权威行业字段。
- 当前聚类中的行业维度来自关键词推断，不是官方行业标签。
- 建议后续引入真实行业列（如申万一级）做分层配额采样。

2) 本地网络环境复杂时易受代理/虚拟网卡影响

- 建议演示时优先使用固定公网入口（云主机或稳定隧道）。
- 本地联调前先用“测试连接”按钮确认可达性。

五、明日接续建议（最小动作）

- 先安装新版 APK，验证设置持久化和测试连接。
- 选股页将条件设为“不限 + 分数>=50”，观察候选前缀分布。
- 对命中标的进入诊股页，确认 matrix 与结构化结果一致。
- 若要长期手机可用，优先迁移后端到低成本云主机。
